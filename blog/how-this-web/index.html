How to Make This Blog

Dec 17 2021

What is this blog?#

This blog is a static website that returns the same hard-coded content from the server whenever a particular resource is requested. A dynamic website, on the other hand, is one where some of the response content is generated dynamically. Learn more about static vs dynamic sites here.

Why do I care?#

This helps provide a concrete introduction to web development that consists of myriad components, including domain acquisition, web file hosting, web file composition (HTML, CSS, JavaScript), and etc. which oftentimes become too many and intimidating to begin diving into.

How to do it?#

We will break down tasks to learn and tackle each step-by-step.

Domain Acquisition#

The idea behind purchasing a domain name is to get your personalized domain, for instance the domain name of this blog is shuhunglin.com. There are several domain name registrars to buy domain names from. In this case, I chose Namecheap because they offer free whois privacy with all their domain names and their pricing is fairly affordable. Learn about other options here.

Web File Composition#

A website consists of several files, such as the HTML, CSS, JavaScript, and image files, which are put together for rendering by web browsers. This YouTube playlist provides a thorough introduction to web development.

There are several ways to create a website. You can create each individual file respectively: HTTL for content of the site, CSS for styling, and JavaScript for adding interactive elements. However, to create a somewhat enriching website, it requires significant amount of work to build from scratch. Luckily, with the open internet, we already have tons of existing web templates to choose from.

Some popular options that provide templates are WordPress and Hugo. Learn more about the pros and cons for each here. Hugo is a static website generator that fits perfectly to our need, where we can use Markdown that makes writing on the web fast and easy.

Web Hosting#

Once we have composed all the necessary web files, e.g. the HTML, CSS, JavaScript files, now we need to host/place the files somewhere in the internet for others to access.

There are several options to host your web files depending on the needs, e.g. static vs dynamic sites. For static websites, learn more about the various options here. We will talk about free hosting via Github Pages.

Now follow the above Github Pages instruction or this YouTube video to have your site setup and link to your purchased domain name.

There you go! You now have completed the full cycle of Domain Acquisition, Web File Composition, and Web Hosting.
